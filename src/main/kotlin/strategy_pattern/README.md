## 主要作用

- 通过定义一系列的算法,把它们一个个封装起来, 并且使它们可相互替换
- 即通过封装，**在不同的场景下提供不同的解决策略**

## 优缺点

### 优点

- **可以方便得切换实现策略**
- 避免了不同情况下的多个 `if` 判断
- **扩展性好**，添加新的策略时，只需继承自对应的 Strategy 接口，并实现相关的方法即可

### 缺点

- **策略类会增多**：因为策略选择时，不是在 `if` 分支里判断选择了，而是单独抽离成类文件了
- 策略类需全部对外暴露：不然外部根本无法使用
